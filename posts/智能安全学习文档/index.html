深度神经网络的对抗攻击及防御方法综述

发表于2025-07-20|更新于2026-03-16|深度学习对抗攻击
|总字数:2.3k|阅读时长:7分钟|浏览量:

引言

深度神经网络（DNN）近年来在诸多领域取得了显著成就，特别是在计算机视觉、自然语言处理和语音识别等方面。然而，研究发现DNN对输入数据的微小扰动极为敏感，这些扰动可以被设计成对抗样本，诱使DNN做出错误决策。这不仅对模型的鲁棒性提出了挑战，也对DNN在实际应用中的安全性带来了严重威胁。本文综述了对抗攻击的各种方法及其对应的防御策略。

一、对抗攻击

对抗攻击是通过在输入数据中加入细微且人眼难以察觉的扰动，使得深度神经网络输出错误结果的攻击方式。根据攻击方法和攻击者掌握的信息，主要分为以下几类：

  1. 基于梯度的对抗攻击：
    • FGSM（Fast Gradient Sign Method）：利用损失函数相对于输入数据的梯度信息，快速生成对抗样本。公式为： $ x_{} = x + (_x J(, x, y)) $
    • BIM（Basic Iterative Method）：是FGSM的迭代版，通过多次小步长的扰动累积生成对抗样本。
  2. 基于优化的对抗攻击：
    • CW攻击（Carlini & Wagner）：通过优化目标函数生成对抗样本，该方法通过调整对抗扰动的大小和攻击成功率之间的平衡，生成最小化扰动的对抗样本。
  3. 基于迁移的对抗攻击：
    • 利用在一个模型上生成的对抗样本攻击另一个模型，这种方法利用了不同模型之间的相似性和对抗样本的迁移性。
  4. 基于生成对抗网络（GAN）的对抗攻击：
    • 通过GAN生成对抗样本，GAN由生成器和判别器组成，生成器生成对抗样本，判别器判断样本是否为真实数据。训练过程中生成器不断优化，生成更具欺骗性的对抗样本。
  5. 基于决策边界的对抗攻击：
    • 通过分析和研究DNN的决策边界，生成能够跨越决策边界的对抗样本，从而成功欺骗模型。

二、对抗攻击防御方法

为了应对对抗攻击，研究者们提出了多种防御策略，这些方法可以分为以下几类：

  1. 基于数据预处理的防御方法：
    • 输入数据的预处理：如对输入图像进行高斯模糊、随机噪声注入、图像压缩等方法，可以在一定程度上减弱对抗样本的影响。
    • 数据增强：通过增加训练数据的多样性，提高模型的泛化能力和鲁棒性。
  2. 增强模型鲁棒性的防御方法：
    • 对抗训练：将对抗样本和正常样本一起用于模型训练，使模型在训练过程中学会识别和抵抗对抗攻击。这是目前较为有效的防御方法之一。
    • 正则化方法：如Dropout、权重衰减等，通过增加模型训练过程中的约束，提高模型的鲁棒性。
  3. 对抗样本检测：
    • 基于统计特征的方法：通过统计分析输入数据的特征，检测对抗样本。例如，分析输入数据的均值和方差。
    • 基于模型不确定性的方法：利用模型对输入数据的不确定性估计，检测可能的对抗样本。
    • 基于独立模型的方法：训练一个专门的检测器，识别和过滤对抗样本，防止其进入主模型。
  4. 模型结构改进：
    • 通过改进模型结构，提高其对对抗样本的抵抗能力。例如，使用更复杂的网络结构、增加模型的层数、引入自注意力机制等。

三、对抗攻击与防御实例

  1. 语义分割：
    • 对抗样本可以使语义分割模型输出错误的分割结果，影响自动驾驶中的目标检测和场景理解。
  2. 音频处理：
    • 在语音识别系统中，对抗样本可以使模型错误地识别语音内容，造成严重后果。
  3. 文本识别：
    • 对抗样本可以使自然语言处理模型输出错误的文本分类或情感分析结果。
  4. 目标检测：
    • 在物体检测任务中，对抗样本可以使DNN错过重要目标或检测出虚假的目标。
  5. 人脸识别：
    • 对抗样本可以使人脸识别系统错误地识别身份，威胁安全系统。
  6. 强化学习：
    • 对抗样本可以干扰强化学习算法的训练和决策过程，导致其输出错误的策略。

四、未来发展趋势

对抗攻击和防御方法的研究仍在不断发展，未来的研究方向可能包括：

  1. 更强的攻击方法：
    • 开发更加隐蔽和有效的对抗攻击算法，特别是针对现有防御方法的改进版本。
  2. 更鲁棒的防御方法：
    • 提高防御方法的泛化能力，使其在不同类型的攻击下都能有效。
  3. 跨领域应用：
    • 将对抗攻击和防御方法应用于更多领域，如金融、医疗等，提升DNN在各领域的安全性。
  4. 理论研究：
    • 深入研究对抗攻击的理论基础，揭示其本质规律，为设计有效防御方法提供理论支持。

白盒攻击详述

白盒攻击是指攻击者对目标模型的结构、参数以及训练数据有完全的了解。这类攻击利用模型的内部信息，通过优化方法找到最小的扰动，使得模型在处理这些扰动后的输入时产生错误输出。

主要白盒攻击方法

  1. FGSM（Fast Gradient Sign Method）

    • 原理：FGSM是一种基于梯度的对抗攻击方法。其基本思想是通过计算损失函数相对于输入的梯度，生成一个方向，使输入在该方向上稍微移动就能最大化损失，从而生成对抗样本。
    • 公式： xadv = x + ϵ ⋅ sign(∇xJ(θ, x, y)) 其中，($ x$ ) 为原始输入，( ϵ ) 为扰动强度，( J(θ, x, y) ) 为损失函数，( ∇xJ ) 为损失函数关于输入 ( x ) 的梯度。
    • 优点：计算简单，生成对抗样本速度快。
    • 缺点：扰动较大时容易被人眼察觉，对某些防御机制的鲁棒性较低。

  2. PGD（Projected Gradient Descent）

    • 原理：PGD是FGSM的多步迭代版本。它通过多次小步的FGSM攻击，逐步优化对抗样本。
    • 过程：
      1. 初始化：( x0 = x )
      2. 迭代更新：( xk + 1 = Projx, ϵ(xk + α ⋅ sign(∇xJ(θ, xk, y))) )
      3. 迭代次数 ( K ) 达到预定值时，停止。 其中，( Projx, ϵ ) 表示将 ( x ) 投影回 ( ϵ ) 范围内。
    • 优点：通过多步优化，可以找到更有效的对抗样本。
    • 缺点：计算开销大，生成对抗样本时间较长。

  3. MI-FGSM(momentum iterative FGSM，MI-FGSM)

    • 基本思想：MI-FGSM 将动量引入到 FGSM 中，通过在每一步迭代中累积梯度的动量，生成更强的对抗样本。其核心思想是在每一步迭代中，利用动量来平滑和加速梯度更新。
    • 公式：

    $$ \begin{aligned} &g_0 = 0, \quad x_{\text{adv}}^0 = x \\ &g_{t+1} = \mu \cdot g_t + \frac{\nabla_x \mathcal{L}(f(x_{\text{adv}}^t), y)}{\|\nabla_x \mathcal{L}(f(x_{\text{adv}}^t), y)\|_1} \\ &x_{\text{adv}}^{t+1} = \text{Clip}_{x, \epsilon}\{x_{\text{adv}}^t + \alpha \cdot \text{sign}(g_{t+1})\} \end{aligned} $$

    • 引入动量好处：
      1. 抑制局部抖动
      2. 加快收敛速度
      3. 增强攻击效果
      4. 提高鲁棒性

  4. CW攻击（Carlini & Wagner攻击）

    • 原理：CW攻击通过优化特定目标函数，直接最小化扰动的同时保证输入被误分类。其目标函数设计巧妙，通常比FGSM和PGD更有效。
    • 目标函数： $ | |_p + c f(x + ) $ 其中，($ $ ) 为扰动，( f(x + δ) ) 为衡量对抗样本 ( x + δ ) 成功误分类的函数，( c ) 为权重。
    • 优点：扰动小，难以被人眼察觉，对防御机制鲁棒性较高。
    • 缺点：优化过程复杂，计算开销大。

白盒攻击的效果和影响

  1. 误分类攻击：攻击者通过白盒攻击生成对抗样本，使模型将其误分类。例如，在图像分类任务中，将“猫”图片误分类为“狗”。
  2. 目标攻击：攻击者有特定目标类，通过白盒攻击使对抗样本被分类为该目标类。例如，将任何输入都分类为“停止标志”。
  3. 泛化攻击：生成的对抗样本不仅能攻击训练模型，还能攻击其他相似模型，提高攻击的泛化能力。

白盒攻击的挑战

  1. 计算复杂性：白盒攻击通常需要计算梯度，优化过程复杂，计算资源消耗大。
  2. 防御机制：模型防御机制的提升，如对抗训练、梯度掩蔽等，会增加攻击难度。
  3. 可解释性：如何解释和理解对抗样本的生成机制以及其对模型的影响仍然是一个研究热点。

通过深入理解白盒攻击及其机制，可以更好地设计防御策略，提高深度神经网络的鲁棒性和安全性。
