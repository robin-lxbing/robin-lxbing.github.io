Cookie基础

发表于2022-03-15|更新于2026-03-16
|总字数:424|阅读时长:1分钟|浏览量:

  • HTTP 协议与 Cookie

    1. Cookie 是由 HTTP 协议制定的，先由服务器保存 Cookie 到浏览器，在下次浏览器请求服务器时把上一次请求得到的 Cookie 归还给服务器
    2. 由服务器创建保存到客户端浏览器的一个键值对，服务器保存 Cookie 的响应头，Set-Cookie: aaa=AAA 。
    3. 当浏览器请求服务器时，会把该服务器保存的 Cookie 随请求发送给服务器，浏览器归还 Cookie 的请求头。
    4. HTTP 协议规定（为浏览器减压）：
      • 一个 Cookie 最大为 4KB；
      • 一个服务器最多向一个浏览器保存 20 个 Cookie
      • 一个浏览器最多保存 300 个 Cookie
    5. 浏览器大战：因为浏览器竞争很激烈，所以很多浏览器都会在一定范围内违反 HTTP 规定，但也不会让一个 Cookie 为 4GB。

  • Cookie 的用途

    1. 服务器使用 Cookie 来识别和跟踪客户端状态。
    2. 保存购物车（购物车中的商品不能使用 request 保存，因为他是一个用户向服务器发送的多个请求）
    3. 显示上交登录名（一个用户的多个请求）

    Cookie不能跨浏览器！

  • Cookie 的 maxAge Cookie 的生命值，即 Cookie 可保存的最大时长，以秒为单位。如：cookie.setMaxAge(60)表示 cookie 会被保存到硬盘 60s。 > - maxAge>0：浏览器会把 Cookie 保存到客户机的硬盘上，有效时长为设置的值。 > - maxAge<0：Cookie 只会在浏览器内存中存在，当用户关闭浏览器时，浏览器进程结束，同时 Cookie 也失去作用。 > - maxAge=0：浏览器会立即删除这个 Cookie。
